# TRIZ Contradiction Analysis

Generate structurally different solutions by framing the problem as a contradiction
and applying TRIZ inventive principles.

## Step 1: Read Vocabulary

Read `${CLAUDE_PLUGIN_ROOT}/techniques/vocabulary.md` for architectural rules.

## Step 2: Dispatch Agent 1 — Contradiction Framer

Use the Agent tool to dispatch a subagent:

**Prompt pattern:**
```
You are a TRIZ analyst. Your job is to frame this problem as a technical
contradiction.

PROBLEM:
{user's problem description}

Identify:
1. What parameter IMPROVES when we take action?
2. What parameter DEGRADES as a result?
3. Map both to TRIZ's 39 engineering parameters (e.g., weight of stationary
   object, speed, force, stability, adaptability, complexity, etc.)
4. From the TRIZ contradiction matrix, identify which of the 40 inventive
   principles are suggested for this parameter pair.

If the problem contains MULTIPLE contradictions, list all of them and identify
the primary one.

Output format:
- Improving parameter: [name] (TRIZ #)
- Degrading parameter: [name] (TRIZ #)
- Applicable inventive principles: [list with numbers and names]
- Contradiction statement: [one sentence]
```

## Step 3: Dispatch Agent 2 — Principle Applicator

Use the Agent tool. This agent receives the problem AND the contradiction output
from Agent 1. It does NOT evaluate feasibility.

**Prompt pattern:**
```
You are a TRIZ solution generator. You have been given a problem and its
contradiction analysis. For EACH applicable inventive principle listed below,
generate ONE concrete solution specific to this problem.

PROBLEM:
{user's problem description}

CONTRADICTION:
{Agent 1's output}

THE 40 TRIZ INVENTIVE PRINCIPLES (reference):
1. Segmentation  2. Taking out  3. Local quality  4. Asymmetry
5. Merging  6. Universality  7. Nested doll  8. Anti-weight
9. Preliminary anti-action  10. Preliminary action  11. Beforehand cushioning
12. Equipotentiality  13. The other way round  14. Spheroidality/Curvature
15. Dynamics  16. Partial or excessive action  17. Another dimension
18. Mechanical vibration  19. Periodic action  20. Continuity of useful action
21. Skipping  22. Blessing in disguise  23. Feedback  24. Intermediary
25. Self-service  26. Copying  27. Cheap disposable  28. Mechanics substitution
29. Pneumatics/hydraulics  30. Flexible shells/thin films  31. Porous materials
32. Color changes  33. Homogeneity  34. Discarding/recovering
35. Parameter changes  36. Phase transitions  37. Thermal expansion
38. Strong oxidants  39. Inert atmosphere  40. Composite materials

Rules:
- ONE concrete solution per applicable principle. Not abstract — specific to
  this problem.
- Do NOT evaluate feasibility. Do NOT rank. Do NOT hedge.
- Include at least one solution that seems impractical — that is often where
  the insight is.
- Label each solution with its principle number and name.
```

## Step 4: Dispatch Agent 3 — Solution Evaluator

Use the Agent tool. This agent receives the problem AND all solutions from Agent 2.
It does NOT pick a winner.

**Prompt pattern:**
```
You are evaluating TRIZ-generated solutions. You have the original problem and
a set of solutions, each generated by a different inventive principle.

PROBLEM:
{user's problem description}

SOLUTIONS:
{Agent 2's output}

Your job:
1. Identify which solutions are STRUCTURALLY DISTINCT from each other (not just
   rephrased versions of the same idea).
2. For each distinct solution, note: does it address the core contradiction
   directly, or does it sidestep it?
3. Flag the most SURPRISING element — the solution or partial solution that
   would be hardest to reach through conventional thinking.
4. Note any partial solutions that could be COMBINED to create something
   stronger.

Do NOT pick a winner. Do NOT rank by feasibility. Your job is to surface
what's interesting, not what's safe.
```

## Step 5: Write Artifact

Combine all three agents' outputs into a single markdown artifact. Write to:
`docs/superpowers/innovate/YYYY-MM-DD-<topic>-triz.md`

Structure:
```markdown
# TRIZ Analysis: <topic>

**Date:** YYYY-MM-DD
**Technique:** TRIZ Contradiction Analysis + Inventive Principles

## Contradiction
{Agent 1's contradiction statement and parameter mapping}

## Solutions by Principle
{Agent 2's solutions, labeled by principle}

## Evaluation
{Agent 3's structural analysis, surprises, combinations}
```

## Important

- Agent 2 NEVER sees Agent 3's evaluation criteria. Isolation matters.
- Agent 3 NEVER picks a winner. The user or brainstorming decides.
- If Agent 1 finds no real contradiction, stop and tell the user — TRIZ may
  not be the right technique. Suggest `:morph` or `:transfer` instead.
